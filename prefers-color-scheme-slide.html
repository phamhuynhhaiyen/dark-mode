System Theme Detection

prefers-color-scheme is where automatic dark mode usually starts

What It Gives You

Browser-level preference signal

The browser exposes the user's OS theme preference, so the app can choose a sensible default before adding any custom theme state.

Automatic default

Useful when the user has not chosen a theme yet.

Early decision

Can be applied before first paint to reduce incorrect-theme flashes.

Native support

Works directly in CSS without adding JavaScript first.

CSS example

@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #111827;
    --color-text: #f9fafb;
    --color-surface: #1f2937;
  }
}

What it does not solve

+
Good for detection

It gives you a strong default based on the system setting.

!
Not enough for full theming

User override, persistence, multi-theme support, and design tokens still need app-level logic.

◐

Key Point

System preference is an input signal, not the whole theming architecture.

1. Detect

Read system preference with prefers-color-scheme.

2. Default

Apply a light or dark default theme early.

3. Override

Let users choose their own theme state.

4. Scale

Move to token-based theming for maintainability.

In a real Angular Material app, the root theme class or root attribute activates the light or dark Material theme, while custom components still use app tokens.

ThemeService
setTheme(mode: 'light' | 'dark' | 'system') {
  localStorage.setItem('theme-mode', mode);
  this.modeSubject.next(mode);
  this.applyResolvedTheme();
}
 Why this setup works
  • One source of truth for theme state
  • Chart.js gets a dedicated adapter layer
  • Angular Material stays on its own theming system
  • Custom components consume semantic tokens only
